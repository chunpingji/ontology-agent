# Feature Specification: 离线本地实体抽取（air-gap 默认 + prose 召回）

**Feature Branch**: `008-gliner-ner-extraction`

**Created**: 2026-06-26

**Status**: Draft

**Input**: User description: "实现 docs/NER by GliNER.md —— 在无互联网（air-gap）部署下，把实体抽取引擎从「云端 LLM 默认」翻转为「本地默认」：结构化源走确定性映射，自由文本（Word 正文 / Excel 自由文本列）走本地零样本 NER，补齐当前丢失的 prose 实体召回；云端 LLM 降级为默认关闭的 opt-in 增强。"

>
> 姊妹路线 `docs/ner-medical-bert-extraction.md`（领域 BERT 微调）与本方案互补，可分阶段并存；本期只交付「零样本、即插即用」的本地 NER 路线。

## Clarifications

### Session 2026-06-26

- Q: 云端 LLM 抽取路径是否彻底移除？ → A: **保留为 opt-in（默认关闭）**——air-gap 下永不触发；仅在显式开启且具备网络+凭据的环境作为增强路径。彻底删除依赖留作后续合规决策（见 Assumptions / 决策项）。
- Q: 自由文本列如何声明？ → A: **每抽取配置一份独立的「自由文本列白名单」**——按列显式 opt-in，未列入的列不做 NER（推荐方案，优于 `column_mapping` 哨兵）。
- Q: 本地 NER 引擎缺失/权重缺失时作业如何处理？ → A: **优雅降级**——静默跳过 prose 召回、退回结构化主路径，作业仍成功完成，并记录 WARNING 供部署自检。
- Q: 离线运行（无网络/无 Key）是否算「降级」？ → A: **否，离线是正常态**——不得标记 `degraded`，避免每条候选误标、实时进度误报降级。
- Q: 本地 NER「抽什么字段」从何而来？ → A: **从目标本体类的数据属性派生**——以属性标签作为抽取标签集，无需为每类手工配置。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - air-gap 下零联网完成抽取（去云依赖） (Priority: P1)

部署运维与评估分析师希望平台在**完全无互联网、无云端大模型、无 API Key** 的内网环境中，仍能端到端完成抽取作业：结构化源（Excel 列、Word 表格、数据库反射）通过**确定性映射**产出 IRI 键控的正确结果；Word 表格的中文表头→属性 IRI 映射改由**确定性列映射**完成（替代此前的云端 LLM），不再依赖任何模型或网络。云端 LLM 抽取被降级为默认关闭的 opt-in 增强，在 air-gap 下永不触发。

**Why this priority**: 这是部署可行性的底座——目标环境本就**不可联网**，若抽取链路仍把云端当默认路径，系统在交付环境中根本无法运行或会因等待云端而挂起/误降级。先把链路翻转为「本地默认、零联网」，结构化主路径零回归，即构成可交付的 MVP；prose 召回（US2/US3）是其上的增量价值。

**Independent Test**: 在断网且未配置 API Key 的环境运行一批结构化源（Excel/含表格的 Word/DB）抽取作业，验证：作业成功完成、不挂起、不失败；结构化列/表格行/库表结构产出与联网基线一致（零回归）；Word 表格中文表头按配置映射为属性 IRI；全程无任何外发网络请求；产出候选**不被误标 `degraded`**、实时进度不误报降级。

**Acceptance Scenarios**:

1. **Given** 断网、无 API Key 的环境与一份结构化 Excel，**When** 发起抽取作业，**Then** 作业成功完成，结构化列按确定性映射产出 IRI 键控候选，与联网基线结果一致，且无外发网络请求。
2. **Given** 一份含中文表头表格的 Word 文档与对应列映射，**When** 抽取该文档，**Then** 表格行的中文表头被确定性映射为属性 IRI（未命中映射的表头原样保留），无需任何模型或云端调用。
3. **Given** 云端 LLM 增强默认关闭，**When** 处理任一源，**Then** 系统不发起任何云端调用，且产出候选不带「降级」标记、实时进度不报 `degraded=true`。
4. **Given** 一个显式开启云端增强且具备网络+凭据的环境，**When** 抽取作业运行，**Then** 云端增强作为 opt-in 路径生效；若其返回为空或失败，则回退结构化结果并作业仍成功。
5. **Given** 数据库源，**When** 反射抽取，**Then** 结构反射产出 class/link 候选，全程离线，行为与现状一致。

### User Story 2 - Word 正文 prose 实体进入审核队列 (Priority: P2)

评估分析师希望 SOP/工艺方案等 Word **正文段落**里的实体不再被丢弃：每段 prose 被本地 NER 抽取为**新实例候选**，进入既有对齐复核队列，并携带指向源段落的溯源锚点（`#para`，贴合 007 文档事实源溯源）。抽取产出沿用既有受控词表标注与对齐流程，复核门禁不被削弱——产出的是候选，经分析师确认方成事实。

**Why this priority**: 这是本期补齐的核心**召回缺口**——当前 Word 正文仅有正则抓强制句，prose 实体召回为空，整段业务信息流失。它依赖 US1 已建立的「本地默认、零联网」链路，故次于 P1；交付后即把「正文段落」变成一等实体来源。

**Independent Test**: 取一份含业务实体描述段落的 Word 文档，在离线环境发起抽取，验证：每个被抽取的 prose 实体生成一条 `instance` 候选进入待复核队列；候选携带 `#para` 溯源、对齐结果/分数/分组键，与结构化候选形态一致；既有的条件式「若…则…必须…」→ Action 候选逻辑不受影响、与 prose 召回并存。

**Acceptance Scenarios**:

1. **Given** 一份含实体描述正文段落的 Word 文档，**When** 在离线环境抽取，**Then** 每段 prose 命中的实体生成 `instance` 候选并入待复核队列，溯源指向对应段落（`#para`）。
2. **Given** 同一正文段落同时含强制句与实体描述，**When** 抽取，**Then** 既产出 Action 候选（既有正则）又产出 prose 实例候选（新增本地 NER），二者并存不互斥。
3. **Given** prose 抽取产出的候选，**When** 进入对齐复核，**Then** 候选已经过受控词表标注与对齐，携带对齐结果、匹配分数、分组键，形态与结构化候选一致。
4. **Given** 一段文本中同一属性被多次命中，**When** 聚合该实体记录，**Then** 多次命中聚成多值列表（不丢值、不互相覆盖）。

### User Story 3 - Excel 自由文本列回填本行属性（富化） (Priority: P3)

评估分析师希望 Excel 中的**自由文本列**（如「工艺描述」「备注」「变更说明」）里此前丢失的属性被找回：对在抽取配置中**按列声明**为自由文本的列，本地 NER 抽取结果**并回本行结构化记录**（富化），而非另起新候选。结构化列权威——NER **仅补未填属性**，已填值不被覆盖；未列入白名单的自由文本列不做 NER。

**Why this priority**: 这是对结构化主路径的补完——单元格级、按列 opt-in、低风险富化，把备注/描述列里的漏填属性补回对应行。它建立在 US1（离线链路）与 US2（本地 NER 能力）之上，作用面与风险均小于前两者，故置 P3。

**Independent Test**: 取一份含自由文本列的 Excel，声明该列为 NER 列，在离线环境抽取，验证：自由文本列被 NER 抽取并把结果并回本行；结构化列已填的属性不被覆盖、仅空缺属性被补；未声明为 NER 列的自由文本列不被处理；富化过程不产生额外的新候选。

**Acceptance Scenarios**:

1. **Given** 一份含「工艺描述」自由文本列的 Excel，且该列已声明为 NER 列，**When** 抽取，**Then** 该列文本经本地 NER 抽取，结果并回本行记录，不新增独立候选。
2. **Given** 自由文本列抽出的属性与结构化列冲突，**When** 合并，**Then** 结构化列值权威保留，NER 仅填充结构化列为空/缺失的属性。
3. **Given** 一列自由文本未被声明为 NER 列，**When** 抽取，**Then** 该列不做 NER（默认丢弃或按既有杂项列规则处理），不产生富化。
4. **Given** 本地 NER 不可用，**When** 抽取含自由文本列的 Excel，**Then** 自由文本列原文被丢弃、结构化行照常产出，作业成功（见 US1 降级行为）。

### Edge Cases

- **网络/Key 缺失**：在 air-gap 下属正常态——作业照常完成，候选**不标 `degraded`**，实时进度不报降级。
- **本地 NER 不可用**（依赖未安装 / 权重缺失 / 加载失败）：静默跳过 prose 召回，退回结构化兜底，作业**不失败**，并记 WARNING 供部署自检。
- **首作业冷启动**：通过启动预热把模型加载前移到启动期，避免首作业承受加载延迟；air-gap 下仅本地 I/O。
- **超长文本**：逐段/逐单元格已是天然切块；单段/单元格仍超模型长度上限时需进一步切分，不得截断丢内容。
- **单元格描述多个独立实体**（应拆行的情形）：不在富化范围，保持多值或留待人工拆分；不强行从一格生成多条独立候选。
- **同标签多次命中**：聚成多值列表。
- **Word 表格无列映射**：未命中的中文表头原样保留为键，不阻断产出。
- **未映射杂项列**：默认丢弃，或按白名单纳入；不静默混入结构化记录。
- **运行期出现 stray 联网调用**：在强制离线保障下不得尝试外发或卡在下载重试。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 抽取流水线 MUST 在**完全离线**（无网络、无 API Key）下端到端完成；结构化源（Excel 列、Word 表格、数据库反射）MUST 通过**确定性映射/反射**产出，不发起任何云端或网络调用。
- **FR-002**: 云端 LLM 增强 MUST 为 opt-in 且**默认关闭**；仅当显式开启**且**具备网络+凭据时方可调用；air-gap 下 MUST 永不触发。
- **FR-003**: 离线运行（无云端 LLM）MUST 被视为**正常态**——MUST NOT 将候选标记为 `degraded`，MUST NOT 通过实时进度上报降级；`degraded` 仅保留给「已开启云端增强但调用失败/为空」等真实降级情形。
- **FR-004**: Word 表格行的（中文）表头→属性 IRI 映射 MUST 改由**确定性列映射**完成（与 Excel 列映射一致），替代此前的云端 LLM 映射；未命中映射的表头 MUST 原样保留为键。
- **FR-005**: Word **正文段落**（prose）MUST 经本地 NER 抽取为**新实例候选**进入待复核队列，候选 MUST 携带指向源段落的溯源引用（`#para`）。
- **FR-006**: 既有的条件式「若…则…必须…」→ Action 候选抽取 MUST 保留，并与 prose 实例召回**并存不互斥**。
- **FR-007**: Excel **自由文本列** MUST 支持每抽取配置一份**独立白名单**按列声明；仅声明的列做 NER，未声明的列 MUST NOT 做 NER。
- **FR-008**: 自由文本列的 NER 结果 MUST **并回本行结构化记录**（富化），MUST NOT 产生新候选；结构化列权威——NER MUST 仅补本行**空缺/未填**的属性，MUST NOT 覆盖已填值。
- **FR-009**: 本地 NER「抽取哪些字段」MUST 从**目标本体类的数据属性**派生（属性标签作为标签集），无需逐类手工配置抽取字段。
- **FR-010**: prose 抽取产出 MUST 复用既有受控词表标注与对齐流程，使候选携带对齐结果、匹配分数、分组键，形态与结构化候选一致；复核门禁 MUST NOT 被削弱（产出为候选，须人工确认）。
- **FR-011**: 同一文本中同一属性的多次命中 MUST 聚成**多值列表**（不丢值、不互相覆盖）。
- **FR-012**: 本地 NER 能力不可用（依赖缺失 / 权重缺失 / 加载失败）时 MUST 优雅降级：静默跳过 prose 召回、退回结构化兜底，作业 MUST 仍成功完成，并 MUST 记 WARNING 供部署自检。
- **FR-013**: 本地 NER 模型权重与运行依赖 MUST 在**构建期预置**、运行期**纯本地加载**；运行期 MUST 强制离线，使任何 stray 加载都不尝试外发或卡在下载重试。
- **FR-014**: 本地 NER 模型 MUST 进程级单例加载，并在应用启动时**预热**，以消除首作业冷启动延迟。
- **FR-015**: 本地 NER 推理 MUST NOT 阻塞作业的并发处理（推理与事件循环解耦）。
- **FR-016**: 系统 MUST 提供可配置的**置信度阈值**控制被接受的抽取片段，且该阈值可调以平衡查准/查全。
- **FR-017**: 自由文本列白名单 MUST 作为抽取配置的一部分被持久化（随配置存取）。
- **FR-018**: 既有结构化抽取主路径 MUST **零回归**——仅「默认不再走云」与「prose 新增本地召回」改变行为，结构化产出本身不变。

### Key Entities *(include if feature involves data)*

- **抽取作业 (Extraction Job)**：对一份上传源（Excel/Word/DB）执行抽取的一次运行；本期要求其在离线环境可成功完成。
- **抽取候选 (Extraction Candidate)**：抽取产出，带类别（`instance` / `action`）、溯源引用（如 `#para`）、对齐结果/分数/分组键、复核状态；prose 召回产生 `instance` 候选，Excel 富化不产生新候选。
- **抽取配置 (Extraction Config)**：抽取的参数集合，含确定性列映射、目标本体类、置信度阈值、本地 NER 开关、云端增强开关，以及新增的**自由文本列白名单**。
- **本体类抽取 Schema**：从目标本体类**数据属性**派生的抽取标签集，决定本地 NER「抽什么字段」。
- **自由文本 (Free Text)**：Word 正文段落 / Excel 自由文本列单元格——本地 NER 的输入。
- **本地 NER 模型权重 (Local Model Weights)**：构建期预置、运行期纯本地加载的离线制品；缺失时触发优雅降级。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在**无任何网络连接、无 API Key** 的环境中，结构化源抽取作业 100% 可端到端成功完成（不失败、不挂起），且全程零外发网络请求。
- **SC-002**: 离线抽取作业产出**零「降级」误报**（候选 `degraded` 标记与实时进度降级上报均为 0）。
- **SC-003**: 结构化主路径产出相对联网基线**零回归**——同一批结构化样本的候选集合与键控结果一致。
- **SC-004**: Word 正文段落的实体召回从**0 提升为可量化**——此前不进队列的 prose 实体现进入待复核队列；召回率以真实中文 SOP/方案样本标定并记录（初始目标值在标定后确定）。
- **SC-005**: Excel 自由文本列中此前丢失的属性被回填到对应行——以真实样本衡量「自由文本列补回的属性数 / 应补属性数」并记录，且结构化已填值 0 被覆盖。
- **SC-006**: 本地 NER 不可用时，作业成功率仍为 100%（仅 prose 召回为空），并产生可用于部署自检的 WARNING。
- **SC-007**: 启用预热后，首作业不承受模型冷启动延迟（模型加载在应用启动期完成，首作业加载等待为 0）。

## Assumptions

- **云端 LLM 去留**：本期**保留**云端路径为 opt-in（默认关闭），不彻底删除其依赖；是否最终移除 `anthropic` 依赖与相关代码留作后续合规决策（见下「决策项」）。
- **自由文本列声明方式**：采用**独立配置字段**（自由文本列白名单），而非在 `column_mapping` 中混入哨兵值。
- **受控词表注入**：沿用既有 span 后处理归一化（复用受控词表标注），不在本期切换到结构化抽取式的枚举约束。
- **本地 NER 引擎与权重**：选用零样本、标签驱动、可本地运行的多语 NER 引擎（具体引擎与权重在 `plan.md` 确定）；语义对齐沿用既有本地中文嵌入模型。中文是硬约束——所选权重须支持中文药企文本。
- **权重分发**：模型权重在有网络的构建/打包环节预置，随部署件分发（制品库 / git-LFS / 随镜像，载体待定），并登记校验和；**不入 git**。
- **离线强制**：运行期通过强制离线（本地加载 + 环境隔离）双保险，杜绝任何 stray 联网。
- **阈值标定**：置信度阈值初始取中等值，正文召回偏低时下调；最终值需以真实中文样本做查准/查全标定。
- **依赖既有栈**：复用既有抽取流水线、对齐、受控词表、嵌入对齐与配置体系；新增自由文本列白名单字段需数据库迁移。
- **单格多实体**：一格描述多个应拆行的独立实体不在富化范围，保持多值或留待人工拆分。

### 决策项（留待 plan / 后续）

- 云端 LLM 路径最终是否**彻底移除**（删依赖与代码）取决于合规要求。
- 模型权重**分发载体**（制品库 / git-LFS / 随镜像）与交付清单/校验和流程待定。
- 评估集与阈值标定：需准备中文 SOP/Excel 自由文本样本做查准/查全评估，据以定阈值与 SC-004/SC-005 的目标值。
