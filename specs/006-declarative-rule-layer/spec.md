# Feature Specification: 推理引擎规则层声明式化（§8.0 升级路径）

**Feature Branch**: `006-declarative-rule-layer`

**Created**: 2026-06-24

**Status**: Draft

**Input**: User description: "推理引擎规则层声明式化与《临床药物智能辅助生产平台方案》§8.0 升级路径落地——把硬编码在应用代码中的风险分类规则知识外化为本体原生、可版本化的声明式表达，并据 §8.0「概念澄清/升级路径」让现仅可断言或不可表达的风险分类类变为可推断，同时保持每条规则的法规可追溯性（GMP）。"

## Clarifications

### Session 2026-06-24

- Q: 声明式判据/规则与冲突策略的存储归属应如何界定？ → A: **TTL + 扩展可编辑 T-Box 元数据表**——以权威 TTL 为真理来源，同时把类表达式/声明式规则投影进可编辑 T-Box 元数据存储（需扩展现仅建模 subClassOf/domain/range/标签/注释的元数据模型），使授权角色经既有编辑界面可维护。
- Q: 激素类/青霉素类"可推断"的充分条件应以哪类本体信号为基？ → A: **外部对齐物化**——以权威外部词表（ChEBI/ATC）对齐条目物化的活性成分类别为判定基，复用既有外部对齐机制（与肿瘤药物 ATC/ChEBI 对齐同机制），不依赖人工布尔标记。
- Q: 零回归（FR-012/SC-004）的判定口径如何界定？ → A: **接受 OWA 改进差异**——逐用例分类/决策级一致；其中"原（因缺省）误判为否 → 现为未知"记为预期改进而非回归。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 风险分类由底层属性自动推断且自带溯源 (Priority: P1)

评估分析师录入药物及其活性成分的客观属性（毒性特征、职业暴露带 OEB、致敏级别、结构标记），无需再手工断言"这是细胞毒/高活性/高致敏/β-内酰胺药物"。系统据这些属性**自动推断**出对应的风险分类，并在评估结果中给出该结论的**判定依据与法规出处**。

**Why this priority**: 这是 §8.0「升级路径」的机制基座——把分类从"靠人记得标"变成"据事实自动得出"。一旦底层属性补录，分类即自动点亮，直接降低漏判这一安全风险；且它独立可用、可端到端验证，足以构成 MVP。

**Independent Test**: 构造一个只含属性、未断言任何风险分类的药物 ABox（如"活性成分具基因毒性特征"），运行评估，验证系统推断出 CytotoxicDrug 并在结果中给出"据 CFDI 2023-03 §3.2"之类的溯源；与现有规则引擎对同一输入的结论一致。

**Acceptance Scenarios**:

1. **Given** 某药物的活性成分标注了基因毒性特征、未断言任何风险分类，**When** 运行风险评估，**Then** 系统推断该药物为细胞毒药物，并附判定依据与法规出处。
2. **Given** 某活性成分的 OEB 为 4，**When** 运行评估，**Then** 系统推断为高活性药物。
3. **Given** 某药物致敏级别为 4（>3），**When** 运行评估，**Then** 系统推断为高致敏药物。
4. **Given** 某活性成分带 β-内酰胺环结构标记，**When** 运行评估，**Then** 系统推断为 β-内酰胺药物。
5. **Given** 同一批历史评估用例，**When** 用声明式判据重算，**Then** 分类结论与原引擎一致（除本特性新增的预期推断外无差异）。

---

### User Story 2 - 闭合"仅可断言/不可表达"缺口（激素/青霉素/肿瘤药物） (Priority: P2)

§8.0 现状表中，激素类、青霉素类只能"人工断言"，肿瘤药物本体里**根本无法表达**。本故事为激素类、青霉素类补上充分条件使其**可推断**，并新增"肿瘤药物（AntineoplasticDrug）"风险类别、对齐权威外部词表，使其可被表达且可推断。

**Why this priority**: 这是 §8.0「升级路径」白纸黑字点名的三个缺口。闭合后，平台覆盖的可推断风险类别从 4 类增至 7 类，消除三处已知漏判面。依赖 P1 建立的推断机制，故次于 P1。

**Independent Test**: 分别构造激素类、青霉素类、肿瘤类药物的 ABox（仅含可判定其类别的底层信号），运行评估，验证三者各自被推断为 HormonalDrug / PenicillinDrug / AntineoplasticDrug，且肿瘤药物类带有对权威外部词表（ATC L01 / ChEBI）的对齐。

**Acceptance Scenarios**:

1. **Given** 某药物携带可判定"激素类"的底层信号，**When** 运行评估，**Then** 推断为激素类药物（不再依赖人工断言）。
2. **Given** 某药物携带可判定"青霉素类"的底层信号，**When** 运行评估，**Then** 推断为青霉素类药物。
3. **Given** 本体中新增了肿瘤药物类别且某药物满足其判据，**When** 运行评估，**Then** 推断为肿瘤药物，并可追溯到 ATC L01 / ChEBI 对齐来源。
4. **Given** 青霉素类被推断成立，**When** 触发设备专用化判定，**Then** 既有"青霉素必须专用化"结论照常成立（与下游规则正确衔接）。

---

### User Story 3 - 规则知识作为可版本化数据被维护，而非埋在代码里 (Priority: P3)

本体工程师 / 法规事务维护者能把分类判据、决策规则（设备专用化、场景识别、污染路径）以及冲突消解策略，作为**声明式、可版本化的制品**阅读和修改——调整一个阈值、补一条判据、改一处安全优先策略，**无需改动应用源码、无需重新部署**，且每条规则仍自带法规出处。

**Why this priority**: 把"规则即代码"改为"规则即数据"，让领域/法规专家可维护、可审计，是本特性的架构性收益；但它是对 P1/P2 已交付能力的泛化与治理增强，故置于 P3。

**Independent Test**: 在声明式制品中调整一条判据（如把致敏阈值从 >3 改为 >2）或新增一条决策规则，不触碰任何应用源码即重算评估，验证行为按预期改变，且变更可在版本历史中追溯到 actor 与批次。

**Acceptance Scenarios**:

1. **Given** 维护者仅编辑声明式判据制品改动一个阈值，**When** 重新发布并重算，**Then** 评估行为按新阈值改变，且全程未修改应用源码。
2. **Given** 多条决策规则对"是否专用化"给出相互矛盾的结论，**When** 系统据声明式冲突策略聚合，**Then** 安全优先（要求专用化）结论胜出。
3. **Given** 任一规则被变更，**When** 查看审计与版本记录，**Then** 可追溯到 actor、批次（TTL 导出 + Git 提交）与时间。

---

### Edge Cases

- **属性缺失（开放世界）**：当判定某分类所需的底层属性未被录入时，该分类的状态 MUST 为"未知"，MUST NOT 被推断为"否"——系统不得断言负类，也不得据"未提及"做出放行结论。
- **结论冲突**：当一条规则结论"允许共线"而另一条结论"必须专用化"时，安全优先结论 MUST 胜出。
- **未建模三元组保全**：写回权威本体时，既有的注释、外部对齐与既有/新增声明式规则等"未被结构化建模"的内容 MUST 被逐字保留，不得被静默剥离。
- **判据成立但下游缺数据**：分类被推断成立、但下游（如 MACO 或污染评分）所需数据缺失时，系统 MUST 给出明确的"数据不足"提示而非静默跳过。
- **同义/重叠判据**：同一药物同时满足多个分类判据时，MUST 全部推断成立（分类非互斥，除非本体显式声明互斥）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 当药物或其活性成分的底层属性满足某风险分类的充分条件时，系统 MUST 自动推断出该分类（细胞毒 / 高活性 / 高致敏 / β-内酰胺），无需人工断言类成员。
- **FR-002**: 每个被推断的分类 MUST 携带其判定依据（触发的判据标识 + 输入事实 + 法规出处），可在评估结果中查看与逐条追溯。
- **FR-003**: 风险分类的判定知识 MUST 以声明式、可版本化、本体原生的制品表达，可在**不修改应用源码、不重新部署**的前提下变更并生效。该判定知识 MUST 以权威 TTL 为单一真理来源，并 MUST 同时投影进平台**可编辑 T-Box 元数据存储**——为此 MUST 扩展现有 T-Box 元数据模型以容纳类表达式与声明式规则（当前仅建模 subClassOf/domain/range/标签/注释），使授权维护者经既有编辑界面可视化维护判据/规则/策略；双存储 MUST 写后一致（宪章 II）。
- **FR-004**: 数值阈值类判据（致敏级别 > 3、OEB 为 4 或 5）MUST 由声明式判据表达并参与推断，不得以代码常量形式硬编码。
- **FR-005**: 系统 MUST 为活性成分引入"β-内酰胺环结构标记"属性，作为 β-内酰胺分类充分条件的输入信号。
- **FR-006**: 激素类药物与青霉素类药物 MUST 从"仅可断言"升级为"可推断"——其充分条件 MUST 以权威外部词表（ChEBI/ATC，如激素类、青霉素类对齐条目）物化的活性成分类别为判定基，复用既有外部对齐机制（非受管谓词，可安全穿过外科式合并），不依赖人工布尔标记。
- **FR-007**: 系统 MUST 新增"肿瘤药物（AntineoplasticDrug）"风险类别，使其在本体中可被表达，对齐权威外部词表（ATC L01 / ChEBI），并具备可推断判据。
- **FR-008**: 超出"充要条件定义"表达力的决策/产生式规则（设备专用化、场景识别、污染路径）MUST 以声明式规则表达，并各自携带法规出处。
- **FR-009**: 冲突消解策略（安全优先覆盖、风险等级取最严）MUST 外化为声明式配置，其优先次序与覆盖方向可被审阅与调整，而非埋于代码分支。
- **FR-010**: 在开放世界假设下，判定属性缺失 MUST 表现为分类"未知"，系统 MUST NOT 据此推断为"否"或断言负类。
- **FR-011**: 对存在相互矛盾结论的情形，安全优先结论（要求专用化）MUST 覆盖宽松结论（允许共线）。
- **FR-012**: 迁移 MUST 保持与现有规则引擎的结论一致性：对既有评估回归用例，除本特性新增的预期推断外，结论 MUST NOT 发生非预期改变（零回归）。
- **FR-013**: 对权威本体的任何写入 MUST 以外科式合并回写，逐字保留未建模三元组（注释、外部对齐、既有/新增声明式规则），且写入前 MUST 提供三元组级 diff 预览。（宪章 II）
- **FR-014**: 新增的定义判据与规则 MUST NOT 破坏既有 BFO 上层对齐与外部对齐，且 MUST 通过模型一致性 / 健康度校验后方可发布；阻断性问题 MUST 拦截发布。（宪章 IV）
- **FR-015**: 规则知识的每次变更 MUST 可追溯（版本、actor、批次、时间），并随发布以批次（一次 TTL 导出 + 一次 Git 提交，SHA 归档）落档。（宪章 III）
- **FR-016**: 规则知识的编辑与发布 MUST 受角色门禁约束，仅授权角色可变更并发布判据/规则/策略。（宪章 安全与合规）

### Key Entities *(include if feature involves data)*

- **风险分类判据 (Classification Criterion)**：把"底层属性条件 → 风险分类"显式化的声明式单元。关键属性：判据标识、触发条件、目标分类、法规出处、逻辑地位（充要定义型 / 产生式）。
- **决策规则 (Decision Rule)**：对设备专用化 / 场景 / 污染路径等给出结论的声明式规则。关键属性：规则标识、前件条件、结论、法规出处、优先级。
- **冲突消解策略 (Conflict-Resolution Policy)**：把多条相互矛盾结论聚合为单一结论的声明式策略。关键属性：适用维度（专用化 / 风险等级）、优先格、覆盖方向（安全优先）。
- **风险分类类 (Risk Classification Class)**：受管本体类，含本特性新增的肿瘤药物类。关键属性：标签、充分条件判据、外部词表对齐。
- **推断溯源记录 (Derivation Provenance)**：单次推断的证明链。关键属性：被推断分类、触发判据、输入事实、法规出处、批次/版本指针。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 可由系统自动推断的风险分类类别数从当前的 4 类提升至 ≥ 7 类（新增激素、青霉素、肿瘤三类的可推断能力）。
- **SC-002**: 对底层属性齐备的药物，目标分类 100% 由系统自动推断得出，分析师手工断言类成员的次数为 0。
- **SC-003**: 分类判据、决策规则与冲突策略的变更 100% 可仅通过编辑声明式制品完成；需要改动应用源码才能调整规则的占比为 0%。
- **SC-004**: 对既有评估回归用例集，非预期的结论变化数为 0（零回归）；所有结论差异均可归因于本特性预先声明的新增推断。
- **SC-005**: 每条被推断的结论 100% 附带可追溯到具体法规出处的判定依据。
- **SC-006**: 判定属性缺失的场景 100% 表现为"未知"，负类误断（把"未知"当作"否"）数为 0。
- **SC-007**: 每次规则知识变更 100% 可在审计与版本历史中追溯到 actor、批次与时间。

## Assumptions

- **判据归属**：声明式判据/规则以权威本体（TTL）为真理来源并随之版本化发布，**同时投影进平台可编辑 T-Box 元数据存储**使其经既有编辑界面可维护（见 FR-003、澄清会 2026-06-24 决策 1）；推断由平台既有的本体推理能力承担（不引入并行推理框架，宪章 V）。
- **激素/青霉素判定信号**：其充分条件以**权威外部词表对齐（ChEBI/ATC）物化的活性成分类别**为判定基（与肿瘤药物 ATC/ChEBI 对齐同机制；见 FR-006、澄清会 2026-06-24 决策 2），不依赖人工布尔标记。
- **肿瘤药物对齐**：AntineoplasticDrug 经 ATC=L01 / ChEBI（如 CHEBI:35610 抗肿瘤药）物化对齐，复用既有外部对齐机制（非受管谓词，可安全穿过外科式合并），不为此新引入对齐框架。
- **零回归口径**：以"逐用例分类/决策级一致"为准；其中 OWA 语义带来的"原误判为否 → 现为未知"视为**预期改进**而非回归（见澄清会 2026-06-24 决策 3）。
- **回归基线**：FR-012 / SC-004 依赖一份既有评估用例的结论基线快照；若尚不存在，本特性 MUST 先据现行引擎建立该基线。
- **范围非目标**：不改前端的**整体布局/导航与评估流程 UI**；不重写 MACO 计算。但因决策 1 要求判据/规则经 UI 可编，**为新增声明式制品提供的最小编辑入口属在范围内**（其具体形态于 plan 阶段定）。外部本体对齐**限于**支撑可推断判据所必需者——肿瘤药物（ATC L01 / ChEBI:35610）以及激素类、青霉素类的 ChEBI/ATC 对齐（决策 2）；不为其它目的新增外部对齐。

### Resolved Decisions *(已于澄清会 2026-06-24 敲定，见 [Clarifications](#clarifications))*

1. **声明式判据/规则的存储归属** → **TTL + 扩展可编辑 T-Box 元数据表**：以权威 TTL 为真理来源，同时把类表达式/声明式规则投影进可编辑 T-Box（需扩展现仅建模 subClassOf/domain/range/标签/注释的元数据模型）使其经 UI 可编。范围相应扩大（含元数据模型扩展与最小前端编辑入口）。
2. **激素/青霉素充分条件取值** → **ChEBI/ATC 外部对齐物化**：以外部词表对齐条目物化的活性成分类别为判定基，与肿瘤药物对齐同机制。
3. **零回归判定口径** → **接受 OWA 改进差异**：逐用例分类/决策级一致；"否 → 未知"记为预期改进而非回归。

### Dependencies

- 平台既有的本体推理能力与权威 TTL 写回（外科式合并 + 三元组级 diff）机制。
- 既有评估回归用例集 / 结论基线（用于零回归验证；缺失则需先建立）。
- 权威外部词表 ATC / ChEBI 的可引用条目（用于肿瘤药物、激素类、青霉素类对齐）。
- 可编辑 T-Box 元数据模型的扩展能力（现仅建模 subClassOf/domain/range/标签/注释，需扩展以承载类表达式与声明式规则，并支撑其经 UI 编辑）。
