# Quickstart: 导航信息架构重构 — 端到端验证

验证重构后的分层导航、子 Tab、应用分析合并、独立审批中心与旧链接重定向。判据可执行,逐条对应 spec 的 US/FR/SC。

## 前置

```bash
# 启动整栈(nginx 边缘同源 /api;本机 8081 见 docker-compose.override.yml)
docker compose up -d --build
# 应用地址:http://localhost:8081   (规范部署为 http://localhost)
```

仅前端迭代时:
```bash
cd frontend && npm install && npm run dev   # http://localhost:3000
```

## 质量门禁(实现完成后必跑)

```bash
cd frontend
npm run lint        # 0 error
npm run build       # standalone 构建通过(含 redirects() 校验)
```
后端不在本特性改动范围 —— 既有 pytest 不需为此特性新增/修改。

## 场景 1 — 分层导航与总览(US1 / FR-001,002,010,012,013)

1. 打开应用 → 左侧侧栏 MUST 显示 **5 个顶层入口**:总览、本体工作台、图谱管理、事实源、审批中心(审批中心是否可见取决于角色,见场景 3)。
2. "图谱管理"下 MUST 见 2 子项:实体管理、应用分析。
3. 顶层 MUST NOT 出现 "文档抽取""推理控制台""知识图谱" 独立项。
4. 点击 **总览** → 进入 `/overview` 落地页(非仅 Logo 跳转);页面显示 KG 统计等概览。
5. 进入实体管理 → "图谱管理"组与"实体管理"子项 MUST 同时高亮。

**Pass**: 顶层 5 项、子项 2 个、无旧平铺项、总览可达、高亮正确。

## 场景 2 — 文档抽取归入实体管理(US2 / FR-004,005;SC-003)

1. 导航 图谱管理 › 实体管理 → 默认在"实体浏览/检索"Tab(`/entities`)。
2. 点"文档抽取"子 Tab → URL 变为 `/entities/extraction`(≤1 次点击可达,SC-003)。
3. 新建抽取作业 → 实时进度可见 → 候选复核(合并/拆分)→ 持久化为实体;全程不离开实体管理分区。
4. **流不丢失**:作业进行中(进度流未终态)时切到"实体浏览"Tab 再切回 → 仍可继续查看该作业进度(经 `activeJobId` 持久化重订阅)。
5. **深链**:直接在地址栏输入 `/entities/extraction` → 正确落在文档抽取 Tab。
6. **前进/后退**:在两子 Tab 间切换后用浏览器后退 → 回到上一个子 Tab。

**Pass**: 子 Tab 一级可达、全流程功能等价、进行中流不丢、深链与前进后退正确。

## 场景 3 — 独立审批中心与角色门禁(US3 / FR-007,008,009;SC-002)

> 用侧栏底部**身份切换器**切换角色(复用 `setIdentity`)。

**3a 非 qa 不可见/不可执行(FR-009 / US3 AS5)**
1. 角色切到 `senior_analyst` → 侧栏 MUST NOT 出现"审批中心"。
2. 地址栏直达 `/approvals` → 见"需 qa 角色"门禁占位,治理操作不可执行(后端对治理端点亦 403)。

**3b qa 工作台(FR-007,008;SC-002)**
3. 角色切到 `qa` → "审批中心"出现在顶层。
4. 点"审批中心"→ 进入 `/approvals`,见**待签结论列表**(`signatures/pending`)。从进入平台到此 ≤2 次点击(SC-002)。
5. 选一条待签结论 → **电子签批**(重认证用户名/密码 + 含义)→ 结论流转生效、列表移除该项。
   - 重认证密码 = 后端 `qa_reauth_secret`(默认 `qa-reauth`)。
6. 另选一条 → **QA 拒绝**(重认证 + 原因)→ 结论流转拒绝终态、其被抑制动作作废(`voided_actions` > 0 时返回计数)。
7. 触发**审计链验真** → 显示链连续性结论(`ok`);可查看审计列表(`/compliance/audit`)。

**Pass**: 非 qa 不可见/不可执行;qa 可在单页完成 签批 / 拒绝 / 验真,≤2 次点击到达。

## 场景 4 — 应用分析整合(US4 / FR-006)

1. 导航 图谱管理 › 应用分析(`/analysis`)。
2. "推理"Tab:执行一次 PDE 或 MACO 计算、一次风险评估 → 结果与重构前等价;可见规则/结论。
3. "图谱查询"Tab:执行一次 SPARQL 查询、查看 KG 统计 → 与重构前等价。
4. 确认本页**不再**内嵌 QA 签批弹窗(签批已移至审批中心;可留"前往审批中心"链接)。

**Pass**: 推理与图谱查询在同一分区可达且功能等价;签批入口已外移。

## 场景 5 — 旧链接重定向(FR-011;SC-005)

逐一在地址栏直接命中,MUST 重定向、MUST NOT 404:

| 输入旧地址 | 期望落点 |
|------------|----------|
| `/extraction` | `/entities/extraction` |
| `/reasoning` | `/analysis` |
| `/knowledge-graph` | `/analysis` |
| `/` | `/overview` |

```bash
# 命令行核验 308(经 nginx 边缘):
curl -sI http://localhost:8081/reasoning | grep -i "location\|HTTP/"
```

**Pass**: 4 条旧路径全部 308 到新位置,0 硬 404。
