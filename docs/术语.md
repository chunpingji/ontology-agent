## 使用约定

- **缩写**列为业内通用英文缩写；无标准缩写者留空，以英文全称或代码标识符为准。
- **来源**列标注该术语在项目内的出处（见下方图例）；区分**文档原文**与**引用标准补充**。
- **✦ 标记**（出现在英文全称旁）：表示该**英文全称系引用标准/通用展开补充，项目文档仅使用缩写、并未拼出全称**——非文档原文，据国际/法规标准补入，便于检索与跨文档对齐。
- **⚠ 歧义提示**：同一缩写在不同语境下含义不同时，在描述中标注（如 `API` 既指「活性药物成分」也指「应用程序接口」）。
- 代码标识符（如 `OntologyEngine`、`ReasoningExecution`）以反引号标注，对应后端 `backend/app/` 实际类/模块。
- 新增术语请按所属**分类**追加至对应小节，并在文末「更新日志」登记。

## 4. SLPRA 领域本体模块

> 命名空间根：`https://ontology.pharma-gmp.cn/slpra/`，7 个模块 + 1 集成桥接，对应平台 Class 分组。来源：模型 / 方案 / TTL。

| 模块 | 英文 / 分组 | 中文全称 | 中英文对照描述 |
|------|-------------|----------|----------------|
| M1 | Drug Product Risk Attributes (`drug`) | 药品风险属性本体 | DrugProduct 及 11 子类、API 三型、Residue、OEBClassification、灭活处置等约 20 个风险属性（~180 公理）。 |
| M2 | GMP Equipment Attributes (`equipment`) | GMP 设备属性本体 | Equipment 树、ConstructionMaterial（材质）、EquipmentSurface（表面）、CleanabilityRating（可清洁性）（~120 公理）。 |
| M3 | Contamination Pathways (`contamination`) | 污染途径模型 | 4 种互斥途径：ResidueContamination、MechanicalTransfer、AirborneTransmission、ConfusionEvent（~90 公理）。 |
| M4 | Risk Assessment Rules (`risk`) | 风险评估规则 | RiskAssessmentProcess、MACOCalculation、8×SharedLineScenario、CompatibilityAssessment、风险级别个体（~60 OWL 公理 + SWRL）。 |
| M5 | Cleaning Validation (`cleaning`) | 清洁验证模型 | CleaningProcess、AnalyticalMethod、CleaningValidationProtocol、再验证触发条件（~100 公理）。 |
| M6 | Facility & Environment (`facility`) | 设施与环境模型 | ProductionArea、CleanArea（A/B/C/D 洁净级别）、ProductionRoom、Airlock（气锁）（~70 公理）。 |
| — | Integration Module (`integration`) | 集成桥接模块 | 跨模块属性 + 外部本体对齐（DrOn/ChEBI/ISA-88/IDMP/BFO）+ 样例个体（`slpra-integration.ttl`）。 |

**核心领域类（示例，来源：模型 / TTL）：**

| 英文类名 | 中文 | 说明 |
|----------|------|------|
| DrugProduct | 药品产品 | 成品剂型，承载临床药物角色（DrOn 范式）。 |
| ActivePharmaceuticalIngredient (API) | 活性药物成分 | ⚠ 此处 API = 活性成分，非「应用程序接口」。 |
| CytotoxicDrug | 细胞毒药物 | 高风险药物类型，原则上需专用设施。 |
| HighActivityDrug / HighPotency | 高活性药物 | 由 OEB 等级判定（OEB≥4/OEB5）。 |
| ClinicalTrialDrug | 临床试验药 | 与商业药共线触发情景 1。 |
| Residue / ResidueContamination | 残留物 / 残留污染 | 交叉污染的主要途径之一。 |
| Equipment / EquipmentSurface | 设备 / 设备表面 | M2 核心类；表面材质影响可清洁性。 |
| ProductionArea / ProductionRoom | 生产区域 / 生产房间 | M6 设施类。 |
| ContaminationEvent | 污染事件 | BFO:process 下的过程类。 |
| SharedEquipmentRole | 共用设备角色 | 设备在共线场景中承担的角色。 |

## 5. 药品风险属性与分类

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 | 来源 |
|------|----------|----------|----------------|------|
| OEB | Occupational Exposure Banding ✦ | 职业暴露分级 | 药物活性/危害分级（OEB1–OEB5），OEB≥4 判高活性、OEB5 须设备专用化。（文档用缩写） | 模型/方案 |
| PDE | Permitted Daily Exposure ✦ | 每日允许暴露量 | 毒理学导出的安全暴露阈值（µg/day），MACO 计算首选输入。（文档用缩写） | 模型 |
| MACO | Maximum Allowable Carryover ✦ | 最大允许残留量 | 前一产品可残留进下一产品的最大量；PDE 法 / 1000 剂量法 / 10ppm 三级降级。（文档用缩写） | 模型 |
| API | Active Pharmaceutical Ingredient | 活性药物成分 | 药品中起治疗作用的成分。⚠ 与「应用程序接口 §15」同缩写。Active ingredient. | 研究 |
| — | β-Lactam / Penicillin | β-内酰胺类 / 青霉素类 | 含 β-内酰胺环的高致敏药物；青霉素共线**无条件**强制专用（CFDI 情景 4）。 | 模型 |
| — | Sensitizing / High-sensitization | 致敏 / 高致敏 | 致敏等级 >3 判为高致敏药物。Sensitization. | 模型 |
| — | Cytotoxic / Genotoxic | 细胞毒 / 基因毒性 | API 有基因毒性 → 产品分类为细胞毒药物。Cytotoxic/genotoxic. | 模型 |
| — | Inactivation Disposition | 灭活处置 / 可灭活性 | 细胞毒是否可灭活决定能否有条件共线（情景 2）。Inactivation feasibility. | 模型 |
| HVAC | Heating, Ventilation, and Air Conditioning ✦ | 暖通空调 / 空气净化系统 | 激素类共线要求独立 HVAC（`requiresIndependentHVAC`）。（文档用缩写） | 模型 |
| TSE/BSE | Transmissible / Bovine Spongiform Encephalopathy ✦ | 传染性 / 牛海绵状脑病 | 生物制品共线须做 TSE/BSE 风险评估（情景 5）。（文档用缩写） | 模型 |
| OEL | Occupational Exposure Limit ✦ | 职业暴露限值 | 空气中可接受浓度限值，与 OEB 配套的防护依据。（文档用缩写，英文全称为展开） | 模型/方案 |

| 状态机守卫 | Lifecycle Guard | 状态机守卫 | `reasoning/lifecycle.py` 单一 `LEGAL_TRANSITIONS`，非法跃迁抛 `IllegalTransition`。Single transition authority. |

| QA 拒绝 | QA Reject | QA 拒绝 | `POST /compliance/reject` 转 `rejected` 终态并作废非终态动作。Reject to terminal. |
| 路线 A / B | Route A / Route B | 路线 A / 路线 B | A=务实闭环（已交付）；B=迁移定义数据化为 `OntologyAction`（显式延后）。Pragmatic vs data-driven. |
