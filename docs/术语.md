## 1. 平台与总体能力

|------|----------|----------|----------------|
| SLPRA | Shared-Line Production Risk Assessment Ontology | 临床药物共线生产风险评估本体模型 | 平台的核心领域本体与项目代号；以 BFO 为上层本体、CFDI《药品共线生产质量风险管理指南》为法规依据，建模药品/设备/污染/风险/清洁/设施六大模块。Core domain ontology & project codename. |
| — | Clinical Drug Intelligent-assisted Production Platform | 临床药物智能辅助生产平台 | 本系统全称；基于本体建模，提供三大能力（T-Box 维护、多源抽取对齐、实时事实源 Action 推理）。The overall system. |

## 2. 本体工程基础

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 |
|------|----------|----------|----------------|
| OWL | Web Ontology Language | 网络本体语言 | W3C 本体表示语言；平台采用 OWL 2，经 Owlready2 在内存 World 中承载。Ontology representation language (OWL 2). |
| RDF | Resource Description Framework | 资源描述框架 | 三元组（主-谓-宾）数据模型，本体序列化与知识图谱的底层模型。Triple-based data model. |
| TTL | Turtle (Terse RDF Triple Language) | Turtle 序列化格式 | RDF 的紧凑文本序列化；权威本体以 `ontology/slpra/*.ttl` 存储，发布时经 `ttl_merge` 外科式合并回写。RDF serialization format. |
| T-Box | Terminological Box | 术语盒 / 定义层 | 本体的**定义层**：Class、LinkType、DataProperty、约束公理等。由高级分析师在工作台维护。Schema/definition layer. |
| A-Box | Assertional Box | 断言盒 / 事实层 | 本体的**事实层**：具体个体及其属性断言；事实源物化的落点。Instance/fact layer. |
| IRI | Internationalized Resource Identifier | 国际化资源标识符 | 本体中类/属性/个体的全局唯一标识（URI 超集）。Globally unique identifier for ontology entities. |
| SWRL | Semantic Web Rule Language | 语义网规则语言 | 在 OWL 之上表达 Horn 子句规则；SLPRA 用其表达风险分类/专用化/评分/情景识别四组规则。Rule language atop OWL. |
| SPARQL | SPARQL Protocol and RDF Query Language | RDF 查询语言 | 针对 RDF 图的标准查询语言（递归缩写）。Standard RDF query language. |
| DL | Description Logic | 描述逻辑 | OWL 的形式化逻辑基础；DL 推理器执行一致性检查与分类。Logical foundation of OWL. |
| OBO | Open Biological and Biomedical Ontologies | 开放生物医学本体（铸造厂） | 生物医学本体协作框架，倡导以 BFO 为共享上层本体、跨本体互操作。OBO Foundry collaborative framework. |
| Class | Class | 类 / 实体类型 | T-Box 中的概念类型（如 `DrugProduct`、`Equipment`）。Concept type. |
| LinkType | Object Property / Link Type | 对象属性 / 关系类型 | 类与类之间的关系定义（domain/range、基数、特征、逆属性），如 `hasActiveIngredient`、`locatedIn`。Relationship definition between classes. |
| DataProperty | Data Property | 数据属性 | 类的字面值属性（datatype、单位、受控词表），如 OEB 等级、PDE 数值。Literal-valued attribute. |
| Restriction | Restriction / Axiom | 约束 / 公理 | OWL 限制（some/only/exactly/min/max/互斥/等价）。OWL restrictions & axioms. |

## 3. 外部本体与标准体系

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 |
|------|----------|----------|----------------|
| BFO | Basic Formal Ontology | 基本形式本体 | 上层本体，ISO/IEC 21838-2:2021 国际标准，被 600+ 本体采用；SLPRA 所有类对齐到 BFO 类别（continuant/process/quality 等）。Top-level ontology. |
| DrOn | Drug Ontology | 药物本体 | 基于 BFO 的药物产品本体，关键创新：**药物产品（而非分子）承载临床药物角色**；`DrugProduct ⊑ dron:00000005`。Drug product ontology. |
| ChEBI | Chemical Entities of Biological Interest | 生物相关化学实体（词典） | 小分子实体词典（180,000+ 类）；活性成分对齐 `API ⊑ chebi:23367`。Molecular entity dictionary. |
| DTO | Drug Target Ontology | 药物靶点本体 | 整合 ChEBI/DOID/Protein Ontology 等，引入四级靶点发展水平（TDL：Tclin/Tchem/Tbio/Tdark）。Drug target ontology. |
| DDI | Drug Discovery Investigations Ontology | 药物发现研究本体 | 定义 R&D 阶段实体（检测开发、苗头确认等），不覆盖生产/法规。⚠ 注意区别于「药物相互作用 drug-drug interaction」。R&D-stage ontology. |
| BRIDG | Biomedical Research Integrated Domain Group | 生物医学研究集成领域组 | CDISC/HL7/ISO/NCI/FDA 协作的临床研究语义标准（ISO 14199:2024）。Clinical research semantic standard. |
| OCRe | Ontology of Clinical Research | 临床研究本体 | 以 OWL 2 逻辑导向建模研究设计，与 BRIDG 信息模型方法互补。Logic-oriented clinical research ontology. |
| CTDasRDF | Clinical Trial Data as RDF | 临床试验数据 RDF 化 | 用 OWL+RDF 建模临床试验概念，整合 BRIDG/W3C Time/SDTM；后续演进为 GoTWLD。Linked-data clinical trial modeling. |
| ISA-88 | Batch Control Standard | 批控制标准 | 批生产控制标准；已被形式化为 OWL 本体；`Equipment ⊑ isa88:EquipmentEntity`。Batch control standard. |
| ISA-95 | Enterprise-Control System Integration | 企业-控制系统集成标准 | 企业与控制系统集成标准，与 ISA-88 共同被形式化。Enterprise-control integration standard. |
| MOMO | Manufacturing Operation Management Ontology | 制造运营管理本体 | NIST 提出，跨离散/批次/连续制造的参考本体，从 ISA-88/95 提取术语并与 IOF 对齐（仍为草案）。Reference manufacturing ontology. |
| OntoReg | Ontology-based Regulatory compliance | 本体法规合规系统 | 以 OWL 本体 + SWRL 规则封装制药过程与法规知识，方法论被 SLPRA 双表示法借鉴（系统本身已过时）。OWL+SWRL regulatory reasoning. |
| IDMP | Identification of Medicinal Products | 药品识别标准 | ISO 药品识别标准；`DrugProduct ⊑ idmp:MedicinalProduct`。ISO medicinal product identification. |
| IOF | Industrial Ontology Foundry | 工业本体铸造厂 | 基于 BFO 的工业领域核心本体，MOMO 与之对齐。BFO-based industrial ontology. |
| CDISC | Clinical Data Interchange Standards Consortium | 临床数据交换标准协会 | 临床数据标准组织（SDTM 等）。Clinical data standards body. |
| SDTM | Study Data Tabulation Model | 研究数据制表模型 | CDISC 的临床试验数据表格标准。CDISC tabulation model. |
| HL7 | Health Level Seven | 卫生信息交换标准（七层） | 医疗信息交换标准组织（含 FHIR）。Health information exchange standard. |
| FHIR | Fast Healthcare Interoperability Resources | 快速医疗互操作资源 | HL7 的现代医疗数据交换规范。Modern healthcare interoperability spec. |
| DOID | Disease Ontology | 疾病本体 | 疾病分类本体，被 DTO 集成。Disease classification ontology. |
| TDL | Target Development Level | 靶点发展水平 | DTO 的四级靶点分类：Tclin/Tchem/Tbio/Tdark。Four-tier target classification. |
| SNOMED CT | Systematized Nomenclature of Medicine — Clinical Terms | 医学系统命名法-临床术语 | 大型临床术语系统（文中作为 DrOn 纠正错误分类的反例）。Large clinical terminology. |
| MeSH | Medical Subject Headings | 医学主题词表 | NLM 生物医学主题词表。Biomedical thesaurus. |
| DOLCE | Descriptive Ontology for Linguistic and Cognitive Engineering | 语言与认知工程描述本体 | 认知导向上层本体（制药领域较少用）。Cognitive-oriented upper ontology. |
| SUMO | Suggested Upper Merged Ontology | 建议上层合并本体 | 通用上层本体（制药领域应用有限）。General upper ontology. |

## 4. SLPRA 领域本体模块

> 命名空间根：`https://ontology.pharma-gmp.cn/slpra/`，7 个模块 + 1 集成桥接，对应平台 Class 分组。

| 模块 | 英文 / 分组 | 中文全称 | 中英文对照描述 |
|------|-------------|----------|----------------|
| M1 | Drug Product Risk Attributes (`drug`) | 药品风险属性本体 | DrugProduct 及 11 子类、API 三型、Residue、OEBClassification、灭活处置等约 20 个风险属性（~180 公理）。 |
| M2 | GMP Equipment Attributes (`equipment`) | GMP 设备属性本体 | Equipment 树、ConstructionMaterial（材质）、EquipmentSurface（表面）、CleanabilityRating（可清洁性）（~120 公理）。 |
| M3 | Contamination Pathways (`contamination`) | 污染途径模型 | 4 种互斥途径：ResidueContamination（残留）、MechanicalTransfer（机械转移）、AirborneTransmission（空气传播）、ConfusionEvent（混淆）（~90 公理）。 |
| M4 | Risk Assessment Rules (`risk`) | 风险评估规则 | RiskAssessmentProcess、MACOCalculation、8×SharedLineScenario、CompatibilityAssessment、风险级别个体（~60 OWL 公理 + SWRL）。 |
| M5 | Cleaning Validation (`cleaning`) | 清洁验证模型 | CleaningProcess、AnalyticalMethod、CleaningValidationProtocol、再验证触发条件（~100 公理）。 |
| M6 | Facility & Environment (`facility`) | 设施与环境模型 | ProductionArea、CleanArea（A/B/C/D 洁净级别）、ProductionRoom、Airlock（气锁）（~70 公理）。 |
| — | Integration Module (`integration`) | 集成桥接模块 | 跨模块属性 + 外部本体对齐（DrOn/ChEBI/ISA-88/IDMP/BFO）+ 样例个体（`slpra-integration.ttl`）。 |

**核心领域类（示例）：**

| 英文类名 | 中文 | 说明 |
|----------|------|------|
| DrugProduct | 药品产品 | 成品剂型，承载临床药物角色（DrOn 范式）。 |
| ActivePharmaceuticalIngredient (API) | 活性药物成分 | ⚠ 此处 API = 活性成分，非「应用程序接口」。 |
| CytotoxicDrug | 细胞毒药物 | 高风险药物类型，原则上需专用设施。 |
| HighActivityDrug / HighPotency | 高活性药物 | 由 OEB 等级判定（OEB≥4/OEB5）。 |
| ClinicalTrialDrug | 临床试验药 | 与商业药共线触发情景 1。 |
| Residue / ResidueContamination | 残留物 / 残留污染 | 交叉污染的主要途径之一。 |
| Equipment / EquipmentSurface | 设备 / 设备表面 | M2 核心类；表面材质影响可清洁性。 |
| ProductionArea / ProductionRoom | 生产区域 / 生产房间 | M6 设施类。 |
| ContaminationEvent | 污染事件 | BFO:process 下的过程类。 |
| SharedEquipmentRole | 共用设备角色 | 设备在共线场景中承担的角色。 |

## 5. 药品风险属性与分类

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 |
|------|----------|----------|----------------|
| OEB | Occupational Exposure Banding | 职业暴露分级 | 药物活性/危害分级（OEB1–OEB5），OEB≥4 判为高活性、OEB5 须设备专用化。Hazard banding 1–5. |
| PDE | Permitted Daily Exposure | 每日允许暴露量 | 毒理学导出的安全暴露阈值（µg/day），MACO 计算首选输入。Toxicology-derived safe threshold. |
| MACO | Maximum Allowable Carryover | 最大允许残留量 | 共线生产中前一产品可残留进下一产品的最大量；PDE 法 / 1000 剂量法 / 10ppm 通用限度三级降级计算。Max permitted carryover. |
| API | Active Pharmaceutical Ingredient | 活性药物成分 | 药品中起治疗作用的成分。⚠ 与「应用程序接口 Application Programming Interface」同缩写，按语境区分。Active ingredient. |
| — | β-Lactam / Penicillin | β-内酰胺类 / 青霉素类 | 含 β-内酰胺环的高致敏药物；青霉素共线**无条件**强制专用设备/厂房（CFDI 情景 4）。Beta-lactam / penicillin. |
| — | Sensitizing / High-sensitization | 致敏 / 高致敏 | 致敏等级 >3 判为高致敏药物。Sensitization potency. |
| — | Cytotoxic / Genotoxic | 细胞毒 / 基因毒性 | API 有基因毒性 → 产品分类为细胞毒药物。Cytotoxic/genotoxic classification. |
| — | Inactivation Disposition | 灭活处置 / 可灭活性 | 细胞毒是否可灭活决定能否有条件共线（情景 2）。Inactivation feasibility. |
| HVAC | Heating, Ventilation, and Air Conditioning | 暖通空调 / 空气净化系统 | 激素类共线要求独立 HVAC（`requiresIndependentHVAC`）。Independent air system. |
| TSE/BSE | Transmissible / Bovine Spongiform Encephalopathy | 传染性 / 牛海绵状脑病 | 生物制品共线须做 TSE/BSE 风险评估（情景 5）。Prion-risk assessment. |
| OEL | Occupational Exposure Limit | 职业暴露限值 | 空气中可接受浓度限值，与 OEB 配套的防护依据。Airborne exposure limit. |

## 6. 设备、设施与污染途径

| 术语 | 英文 | 中文 | 中英文对照描述 |
|------|------|------|----------------|
| 4 种污染途径 | Contamination Pathways | 污染途径（互斥四类） | 残留（Residue）、机械转移（Mechanical Transfer）、空气传播（Airborne）、混淆（Confusion）；OWL 不相交公理保证互斥。Four mutually-exclusive pathways. |
| 可清洁性 | Cleanability Rating | 可清洁性评级 | 设备表面易清洁程度，影响残留污染风险。Surface cleanability. |
| 洁净级别 | Clean Area A/B/C/D | 洁净区级别 A/B/C/D | GMP 洁净室分级（A 级最高）。GMP cleanroom grades. |
| 气锁 | Airlock | 气锁 | 隔离不同洁净级别区域的缓冲间。Buffer between clean zones. |
| 专用化 | Dedication / Dedicated Equipment | 设备专用化 | 高风险药物须专用独立设备/设施，不与其他产品共线。Dedicated equipment requirement. |
| 设备-产品相容性矩阵 | Equipment-Product Compatibility Matrix | 设备-产品相容性矩阵 | N-ary 关系建模设备与产品能否共用的决策矩阵。N-ary compatibility matrix. |

## 7. 风险计算与清洁验证

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 |
|------|----------|----------|----------------|
| MACO | Maximum Allowable Carryover | 最大允许残留量 | 见 §5；后端 `reasoning/calculators.py` 实现混合计算。Hybrid MACO computation. |
| PDE | Permitted Daily Exposure | 每日允许暴露量 | 见 §5；PDE 可用时优先于 1/1000 法与 10ppm 法。Preferred MACO input. |
| 1/1000 法 | 1/1000 Minimum Therapeutic Dose | 千分之一最小治疗剂量法 | PDE 不可用但有最小治疗剂量时的 MACO 降级方法。Fallback method 2. |
| 10ppm 法 | 10ppm General Limit | 10ppm 通用限度法 | 无毒理学数据时的最末级通用限度。Fallback method 3. |
| — | Cleaning Validation Protocol | 清洁验证方案 | M5 核心产物；定义采样、分析方法与限度。Cleaning validation protocol. |
| — | Analytical Method | 分析方法 | 清洁验证中检测残留的方法（如 HPLC、TOC）。Residue analytical method. |
| SOP | Standard Operating Procedure | 标准操作规程 | GMP 文件体系的操作规程。Operating procedure document. |

## 8. 法规、合规与审计

| 缩写 | 英文全称 | 中文全称 | 中英文对照描述 |
|------|----------|----------|----------------|
| GMP | Good Manufacturing Practice | 药品生产质量管理规范 | 制药生产的法定质量管理体系，平台合规基线。Pharmaceutical quality standard. |

## 13. 结论工作流与状态机

| 缩写/术语 | 英文 | 中文 | 中英文对照描述 |
|-----------|------|------|----------------|
| ReasoningExecution | Reasoning Execution | 推理执行（结论实例） | 持久化的结论实体，带唯一标识与生命周期状态；流水线自举源头。Persisted conclusion instance. |
| ActionExecution | Action Execution | 动作执行 | 结论编排出的动作（工单/告警/排期阻断/建议性回写）；可被抑制（suppressed）或作废（voided）。Orchestrated action record. |
| 生命周期状态 | Lifecycle State | 生命周期状态（四态） | `pending_signature`（待签）/`effective`（生效）/`superseded`（被取代）/`rejected`（拒绝）；后两者为终态。Four-state lifecycle. |
| 状态机守卫 | Lifecycle Guard | 状态机守卫 | `reasoning/lifecycle.py` 单一 `LEGAL_TRANSITIONS`，非法跃迁抛 `IllegalTransition`。Single transition authority. |
| G1 | Gap 1: Assess-to-Persist | 评估即落库 | `POST /reasoning/assess` 评估即落库结论并 orchestrate 动作，使流水线自举。Bootstrap conclusion persistence. |
| G2 | Gap 2: Auto-arm QA Gate | QA 闸门自动布防 | `risk.py` `requires_qa_signature()` 据高风险判据自动置 `pending_signature` 并抑制动作。Auto-arm signature gate. |
| G3 | Gap 3: Auto-recompute Subscriber | 事实变更自动重算 | `recompute_subscriber.py` 订阅 FactEventBus 桥接增量重算，无人工触发。Auto recompute on fact change. |
| 取代链 | Supersession Chain | 取代链 | 增量重算使旧结论 `effective→superseded`，过时结论的待派发动作随之作废。Supersession + action voiding. |
| QA 拒绝 | QA Reject | QA 拒绝 | `POST /compliance/reject` 转 `rejected` 终态并作废非终态动作。Reject to terminal state. |
| 路线 A / B | Route A / Route B | 路线 A / 路线 B | A=务实闭环（已交付）；B=迁移定义数据化为 `OntologyAction`（显式延后）。Pragmatic vs data-driven routes. |
| 回写（建议性） | Advisory Write-back | 建议性回写 | 对外回写为建议性、非权威；`not_accepted` ≠ 失败。Advisory, non-authoritative write-back. |
