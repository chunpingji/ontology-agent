## 1. 设计概述

### 1.1 解决的核心问题

临床药物生产中的**共线生产（Shared-Line Production）**面临交叉污染风险评估的信息集成挑战。评估需要同时考虑：

- **药品维度**：药品的毒理学分类、致敏性、活性、灭活方法等约20个风险属性
- **设备维度**：设备材质、可清洁性、洁净区级别、接触面积等属性
- **过程维度**：4种污染途径的风险概率和严重性
- **法规维度**：8种特殊共线情景的法定要求

现有本体生态中**没有**覆盖这一交叉领域的本体：
- DrOn/ChEBI：覆盖药物产品和化学实体，不涉及生产设备
- MOMO/ISA-88：覆盖制造运营，不涉及药品风险属性
- OntoReg：展示了OWL+SWRL法规推理方法论，但系统已过时

### 1.2 设计原则

| 原则 | 实现方式 | 来源 |
|------|---------|------|
| **模块化** | 7个独立TTL模块 + 1个集成桥接 | ISA-88/95形式化研究(Vegetti & Henning, FOMI 2014) |
| **实在论方法** | 类对应现实实体类型，非概念 | DrOn方法论(Hogan et al.) |
| **角色区分** | 药物产品（而非分子）承载临床药物角色 | DrOn关键创新 |
| **双表示法** | OWL本体(知识) + SWRL规则(推理) | OntoReg方法论 |
| **BFO对齐** | 所有类对齐到BFO类别 | OBO Foundry最佳实践 |

### 3.4 建模说明

**关键设计决策**：药品分类（CytotoxicDrug等）同时作为**类**（通过OWL subsumption推理）和**布尔属性**（通过SWRL规则快速查询）存在。这是有意的冗余设计：

- 类层次支持本体推理器的自动分类（如：一个API有GenotoxicityProfile → 其产品被推理为CytotoxicDrug）
- 布尔属性支持SWRL规则的高效模式匹配和外部系统的简单查询

## 5. Module 3：污染途径模型

### 5.1 类层次

```
BFO:process
└── slpra-contam:ContaminationEvent          # 污染事件 (BFO:process)
    ├── slpra-contam:ResidueContamination    # 途径1：残留污染
    │   # 必要条件：涉及产品接触面
    │   # OWL约束：∃involvesSurface.ProductContactSurface
    │
    ├── slpra-contam:MechanicalTransfer      # 途径2：机械转移
    │   # 必要条件：涉及非接触面或人员
    │   # OWL约束：∃involvesSurface.NonContactSurface
    │
    ├── slpra-contam:AirborneTransmission    # 途径3：空气传播
    │   # 必要条件：发生在生产区域
    │   # OWL约束：∃occursInFacility.ProductionArea
    │
    └── slpra-contam:ConfusionEvent          # 途径4：混淆
        ├── slpra-contam:LabelMixup          # 标签混淆
        ├── slpra-contam:MaterialMixup       # 物料混淆
        └── slpra-contam:DocumentMixup       # 文件混淆

# 不相交公理：4种途径互斥
owl:AllDisjointClasses(
    slpra-contam:ResidueContamination,
    slpra-contam:MechanicalTransfer,
    slpra-contam:AirborneTransmission,
    slpra-contam:ConfusionEvent
)
```

### 5.2 建模说明

**为什么用子类而非属性值？**

4种污染途径作为`ContaminationEvent`的不相交子类建模，而非用一个`hasPathwayType`属性值来区分。原因：

1. **每种途径有不同的因果机制**：残留涉及表面化学，空气传播涉及颗粒力学，混淆涉及流程管理
2. **控制措施不同**：残留→清洁验证，机械转移→人员操作规范，空气传播→HVAC和压差，混淆→文件系统和防差错
3. **SWRL规则可直接按类触发**：无需每条规则都检查属性值

**Pathway 4（混淆）的特殊性**：混淆是过程控制失败，不涉及化学污染，与前三种途径性质根本不同。因此显式声明为不相交类。

### 7.4 清洁验证重新验证触发条件

根据CFDI指南，以下情况需重新验证（建模为SWRL规则）：

- 新产品使共用设备残留限度更严格（更小的MBS或更大的TDD）
- 清洁方法发生变更
- 新产品被识别为最差条件产品

不需要重新验证的情况：
- 清洁方法未变
- 新产品溶解度更高
- 未被识别为最差条件产品

## 10. SWRL规则集

### 10.1 规则组1：药品风险自动分类

```
# R-DC1: API有基因毒性 → 产品分类为细胞毒药物
DrugProduct(?dp) ∧ hasActiveIngredient(?dp, ?api) ∧
    hasToxicityProfile(?api, ?tox) ∧ GenotoxicityProfile(?tox)
    → CytotoxicDrug(?dp)

# R-DC2: OEB ≥ 4 → 分类为高活性药物
DrugProduct(?dp) ∧ hasActiveIngredient(?dp, ?api) ∧
    hasOEBClassification(?api, ?oeb) ∧ (OEB4(?oeb) ∨ OEB5(?oeb))
    → HighActivityDrug(?dp)

# R-DC3: 致敏等级 > 3 → 分类为高致敏药物
DrugProduct(?dp) ∧ hasSensitizationPotential(?dp, ?sp) ∧
    sensitizationLevel(?sp, ?level) ∧ swrlb:greaterThan(?level, 3)
    → HighSensitizingDrug(?dp)

# R-DC4: 含β-内酰胺环结构 → 分类为β-内酰胺类
DrugProduct(?dp) ∧ hasActiveIngredient(?dp, ?api) ∧
    hasBetaLactamRing(?api, true)
    → BetaLactamDrug(?dp)
```

### 10.2 规则组2：设备专用化决策

```
# R-ED1: 青霉素类 → 必须专用设备（无条件，CFDI强制）
DrugProduct(?dp) ∧ PenicillinDrug(?dp) ∧
    usedInProductionOf(?eq, ?dp)
    → requiresDedication(?eq, true)

# R-ED2: 细胞毒 + 不可灭活 → 必须专用
DrugProduct(?dp) ∧ CytotoxicDrug(?dp) ∧
    hasActiveIngredient(?dp, ?api) ∧
    hasInactivationDisposition(?api, ?inact) ∧ NonInactivatable(?inact) ∧
    usedInProductionOf(?eq, ?dp)
    → requiresDedication(?eq, true)

# R-ED3: 生物制品 + 朊病毒风险 → 必须专用
DrugProduct(?dp) ∧ BiologicalProduct(?dp) ∧
    hasPrionRisk(?dp, true) ∧
    usedInProductionOf(?eq, ?dp)
    → requiresDedication(?eq, true)

# R-ED4: 细胞毒 + 可灭活 → 允许共线（有条件）
DrugProduct(?dp) ∧ CytotoxicDrug(?dp) ∧
    hasActiveIngredient(?dp, ?api) ∧
    hasInactivationDisposition(?api, ?inact) ∧
    (HeatInactivatable(?inact) ∨ ChemicalInactivatable(?inact)) ∧
    usedInProductionOf(?eq, ?dp)
    → requiresDedication(?eq, false) ∧
      requiresInactivationValidation(?eq, true)

# R-ED5: 高活性(OEB5) → 必须专用
DrugProduct(?dp) ∧ HighActivityDrug(?dp) ∧
    hasActiveIngredient(?dp, ?api) ∧
    hasOEBClassification(?api, ?oeb) ∧ OEB5(?oeb) ∧
    usedInProductionOf(?eq, ?dp)
    → requiresDedication(?eq, true)

# R-ED6: 激素类 → 独立空气系统
DrugProduct(?dp) ∧ HormonalDrug(?dp) ∧
    usedInProductionOf(?eq, ?dp) ∧
    locatedIn(?eq, ?room)
    → requiresIndependentHVAC(?room, true)
```

### 10.3 规则组3：污染途径风险评分

```
# R-CP1: 残留 + 低PDE + 差可清洁性 → 高风险
ResidueContamination(?ce) ∧
    mediatedByEquipment(?ce, ?eq) ∧
    hasCleanabilityRating(?eq, ?cr) ∧ cleanabilityScore(?cr, ?score) ∧
    swrlb:lessThan(?score, 3) ∧
    hasContaminationSource(?ce, ?dp) ∧
    hasActiveIngredient(?dp, ?api) ∧
    pde_mg_per_day(?api, ?pde) ∧ swrlb:lessThan(?pde, 0.01)
    → hasContaminationSeverity(?ce, slpra-risk:HighRisk)

# R-CP2: 粉末操作 + 共用区域 + 空气传播 → 高风险
AirborneTransmission(?ce) ∧
    hasContaminationSource(?ce, ?dp) ∧
    dosageForm(?dp, "powder") ∧
    occursInFacility(?ce, ?area) ∧
    GeneralArea(?area)
    → hasContaminationSeverity(?ce, slpra-risk:HighRisk)

# R-CP3: 液体操作 + 密闭设备 + 残留途径 → 低风险
ResidueContamination(?ce) ∧
    hasContaminationSource(?ce, ?dp) ∧
    dosageForm(?dp, "solution") ∧
    mediatedByEquipment(?ce, ?eq) ∧
    hasCleanabilityRating(?eq, ?cr) ∧ cleanabilityScore(?cr, ?score) ∧
    swrlb:greaterThan(?score, 4)
    → hasContaminationSeverity(?ce, slpra-risk:LowRisk)

# R-CP4: 相同剂型并行生产 + 混淆途径 → 中风险
ConfusionEvent(?ce) ∧
    hasContaminationSource(?ce, ?dp1) ∧
    hasContaminationTarget(?ce, ?dp2) ∧
    dosageForm(?dp1, ?form) ∧ dosageForm(?dp2, ?form)
    → hasContaminationSeverity(?ce, slpra-risk:MediumRisk)
```

### 10.4 规则组4：CFDI情景自动识别

```
# R-SC1: 临床试验药 + 商业药共用设备 → 情景1
ClinicalTrialDrug(?dp1) ∧ CommercialDrug(?dp2) ∧
    usedInProductionOf(?eq, ?dp1) ∧ usedInProductionOf(?eq, ?dp2)
    → involvesScenario(?eq, slpra-risk:ClinicalWithCommercialScenario)

# R-SC2: 细胞毒药物共线
CytotoxicDrug(?dp) ∧ usedInProductionOf(?eq, ?dp) ∧
    SharedEquipmentRole(?eq)
    → involvesScenario(?eq, slpra-risk:CytotoxicSharedLineScenario)

# R-SC3: 激素类共线
HormonalDrug(?dp) ∧ usedInProductionOf(?eq, ?dp) ∧
    SharedEquipmentRole(?eq)
    → involvesScenario(?eq, slpra-risk:HormonalSharedLineScenario)

# R-SC4: 青霉素共线 → 直接触发专用化
PenicillinDrug(?dp) ∧ usedInProductionOf(?eq, ?dp)
    → involvesScenario(?eq, slpra-risk:PenicillinSharedLineScenario) ∧
      requiresDedication(?eq, true)

# R-SC5: 生物制品共线
BiologicalProduct(?dp) ∧ usedInProductionOf(?eq, ?dp) ∧
    SharedEquipmentRole(?eq)
    → involvesScenario(?eq, slpra-risk:BiologicSharedLineScenario)

# R-SC6: 高活性药物共线
HighActivityDrug(?dp) ∧ usedInProductionOf(?eq, ?dp) ∧
    SharedEquipmentRole(?eq)
    → involvesScenario(?eq, slpra-risk:HighPotencySharedLineScenario)

# R-SC7: 无菌药品共线
SterileDrugProduct(?dp) ∧ usedInProductionOf(?eq, ?dp) ∧
    SharedEquipmentRole(?eq)
    → involvesScenario(?eq, slpra-risk:SterileDrugSharedLineScenario)

# R-SC8: 多剂型共线
DrugProduct(?dp1) ∧ DrugProduct(?dp2) ∧
    dosageForm(?dp1, ?f1) ∧ dosageForm(?dp2, ?f2) ∧
    swrlb:notEqual(?f1, ?f2) ∧
    usedInProductionOf(?eq, ?dp1) ∧ usedInProductionOf(?eq, ?dp2)
    → involvesScenario(?eq, slpra-risk:MultiDosageFormScenario)
```

## 11. MACO/PDE计算模型

### 11.1 混合计算架构

SWRL的算术能力有限（支持`swrlb:multiply`、`swrlb:divide`，但多步链式计算产生过多中间变量）。因此采用**混合架构**：

```
┌────────────────┐     ┌──────────────┐     ┌─────────────────┐
│  OWL本体       │     │ SWRL规则     │     │ 外部计算服务     │
│  (结构关系)    │────▶│ (条件判断    │────▶│ (Python/SPARQL) │
│                │     │  方法选择)   │     │ (算术计算)      │
│ · 输入参数定义 │     │ · 选择计算法 │     │ · PDE公式       │
│ · 类型约束     │     │ · 触发条件   │     │ · MACO公式      │
│ · 结果存储     │     │ · 风险分级   │     │ · min()选择     │
└────────────────┘     └──────────────┘     └─────────────────┘
```

### 11.2 PDE计算公式

```
PDE (mg/day) = (PoD × BW) / (F1 × F2 × F3 × F4 × F5 × MF)
```

| 参数 | 含义 | 本体属性 |
|------|------|---------|
| PoD | 毒理学出发点(mg/kg/day) | `slpra-drug:pod_mg_per_kg_per_day` |
| BW | 体重校正因子(kg)，默认50kg | `slpra-risk:bodyWeightCorrection_kg` |
| F1 | 种属间差异因子 | `slpra-risk:adjustmentFactor_F1` |
| F2 | 个体间差异因子 | `slpra-risk:adjustmentFactor_F2` |
| F3 | 研究周期因子 | `slpra-risk:adjustmentFactor_F3` |
| F4 | 严重毒性因子 | `slpra-risk:adjustmentFactor_F4` |
| F5 | NOAEL/LOAEL因子 | `slpra-risk:adjustmentFactor_F5` |
| MF | 修饰因子 | `slpra-risk:modifyingFactor_MF` |

### 11.3 MACO计算公式

CFDI指南要求从多种方法计算MACO，取**最小值**：

**方法1：1/1000最小治疗剂量法**
```
MACO₁ = (TDD_prev × MBS) / (SF × TDD_next)
SF = 1000
```

**方法2：NOEL法**
```
NOEL = LD50(mg/kg) × BW(kg) / 2000
MACO₂ = (NOEL × MBS) / (SF_route × TDD_next)
SF_route: 外用10-100, 口服100-1000, 注射1000-10000
```

**方法3：10ppm法**
```
MACO₃ = 10ppm × MBS
```

**方法4：PDE法（CFDI首选）**
```
MACO₄ = (PDE × MBS) / TDD_next
```

**最终MACO = min(MACO₁, MACO₂, MACO₃, MACO₄)**

### 11.4 SWRL方法选择规则

```
# R-MACO-1: PDE可用时优先选择PDE法
MACOCalculation(?calc) ∧ hasInputPDE(?calc, ?pde) ∧
    swrlb:greaterThan(?pde, 0)
    → hasMACOMethod(?calc, "PDE-based")

# R-MACO-2: PDE不可用但有最小治疗剂量时用1/1000法
MACOCalculation(?calc) ∧ hasInputPDE(?calc, ?pde) ∧
    swrlb:equal(?pde, 0) ∧
    hasMinTherapeuticDose(?calc, ?td) ∧ swrlb:greaterThan(?td, 0)
    → hasMACOMethod(?calc, "dose-based")

# R-MACO-3: 无毒理学数据时用10ppm通用限度
MACOCalculation(?calc) ∧ hasInputPDE(?calc, ?pde) ∧
    swrlb:equal(?pde, 0) ∧
    hasMinTherapeuticDose(?calc, ?td) ∧ swrlb:equal(?td, 0)
    → hasMACOMethod(?calc, "general-limit")
```

## 12. 设备-产品相容性矩阵

### 12.1 N-ary关系建模

设备-产品相容性是一个**三元关系**（设备 × 产品 → 结果），使用W3C n-ary关系模式建模：

```turtle
slpra-risk:CompatibilityAssessment a owl:Class ;
    rdfs:subClassOf obo:BFO_0000031 ;  # generically_dependent_continuant
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty slpra-risk:assessesEquipmentFor ;
        owl:qualifiedCardinality "1"^^xsd:nonNegativeInteger ;
        owl:onClass slpra-equip:Equipment
    ] ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty slpra-risk:assessesProductFor ;
        owl:qualifiedCardinality "1"^^xsd:nonNegativeInteger ;
        owl:onClass slpra-drug:DrugProduct
    ] .

slpra-risk:compatibilityResult a owl:DatatypeProperty ;
    rdfs:domain slpra-risk:CompatibilityAssessment ;
    rdfs:range [ a rdfs:Datatype ;
        owl:oneOf ("compatible" "conditional" "incompatible") ] .

slpra-risk:compatibilityCondition a owl:DatatypeProperty ;
    rdfs:domain slpra-risk:CompatibilityAssessment ;
    rdfs:range xsd:string .
```

### 12.2 相容性决策逻辑

```
设备材质 × 产品属性 → 相容性判断

SS316L + 强酸/碱药物         → compatible (316L耐腐蚀)
SS304  + 强酸/碱药物         → conditional (需评估腐蚀风险)
SS304+HALAR + 强腐蚀性药物   → compatible (HALAR耐腐蚀涂层)
Glass  + 强碱药物(NaOH>1M)   → incompatible (玻璃被碱腐蚀)
Glass  + 细胞毒药物          → conditional (玻璃易清洗但需验证灭活)
任意设备 + 青霉素类          → incompatible (除非专用设备)
```

### 13.3 风险评估推理链示例

以"细胞毒药物DrugX在共用离心机CT64201上的风险评估"为例：

```
输入事实:
  DrugX : ClinicalTrialDrug, CytotoxicDrug
  API_CompoundX : CytotoxicAPI, OEB5, PDE=0.002mg/day, ChemicalInactivatable
  CT64201 : Centrifuge, SharedEquipmentRole, 304SS+HALAR, 一般区

SWRL推理链:

1. R-DC1触发: GenotoxicityProfile → 确认CytotoxicDrug分类 ✓
2. R-DC2触发: OEB5 → 确认HighActivityDrug分类 ✓

3. R-SC2触发: CytotoxicDrug + SharedEquipment
   → involvesScenario(CT64201, CytotoxicSharedLineScenario)

4. R-SC1触发: ClinicalTrialDrug + (假设同线有CommercialDrug)
   → involvesScenario(CT64201, ClinicalWithCommercialScenario)

5. R-ED4触发: CytotoxicDrug + ChemicalInactivatable
   → requiresDedication(CT64201, false)  # 允许共线
   → requiresInactivationValidation(CT64201, true)  # 但需灭活验证

6. R-ED5触发: OEB5
   → requiresDedication(CT64201, true)  # 冲突! OEB5要求专用
   # 冲突解决：requiresDedication=true 优先（安全原则）

7. R-CP1触发: ResidueContamination + PDE<0.01 + cleanabilityScore=3
   → hasContaminationSeverity = HighRisk

最终评估结果:
  CT64201对DrugX:
  - 情景: CytotoxicSharedLine + ClinicalWithCommercial
  - 设备专用化: 必须专用 (OEB5优先)
  - 残留污染风险: 高
  - 如果无法专用: 需灭活验证 + 增强清洁验证 + PDE法计算MACO
  - MACO = (0.002 × MBS) / TDD_next  (极低残留限度)
```

### 13.4 冲突解决策略

当多条SWRL规则产生冲突结论时（如R-ED4说不需专用，R-ED5说需要专用），采用**安全优先原则**：

```
# 冲突解决元规则
requiresDedication(?eq, true) ∧ requiresDedication(?eq, false)
    → requiresDedication(?eq, true)
# 任何一条规则要求专用，最终结论就是专用
```

这一原则与CFDI指南的"法规合规优先"原则一致。在OWL中通过声明`requiresDedication`为functional property（函数性属性）并设置true优先的推理规则来实现。
