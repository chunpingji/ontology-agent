# 风险评估报告生成方案：从文档抽取到结构化报告的技术实现路径

**Date**: 2026-06-29 | **Status**: 方案论证 | **依赖**: 009-word-table-ner-optimize

---

## 1. 应用场景

在对《原料药 HRS-1234 临床备样生产信息.docx》提取关系图谱后，拿到 HRS-1234 的：

- 基本信息描述（分子式、剂型、给药途径、PDE）
- 共线评估数据（NOAEL、HED、安全系数、日剂量）
- 安全评估风险（人员/生产设备/物料管理/文件/三废处理 五类 HazID）
- 使用设备（642/646 车间设备清单，含编号、名称、规格、材质）
- 合成路线（Suzuki 偶联 → 脱保护 → 成盐 → 酰胺缩合 → 脱苄基 → 精制）
- 存放条件（包装方式、温度、避光、有效期）
- 给药途径（口服速释片剂）

结合声明式规则（DecisionRule），自动生成符合 QS-A-020F05 格式的结构化风险评估报告。

## 2. 现有能力盘点

### 2.1 文档结构解析

`parse_docx_structure()`（`backend/app/services/extraction/docx_structure.py:100`）将 `.docx` 解析为 `DocStructure`：

```
DocStructure
├── title: str                      # 首个一级标题
├── sections: [DocSection]          # 按标题切分的章节（heading + paras）
├── tables: [DocTable]              # 表格（headers + rows + cells + header_sig）
├── paragraphs: [str]               # 全部非空段落（扁平）
└── headings: [str]                 # 全部标题（扁平）
```

`DocTable.has_headers(*needles)` 通过**表头签名**匹配定位目标表格（而非脆弱的"表N"题注），`DocStructure.find_section(*needles)` 按标题子串定位章节。

### 2.2 文档级分类

`document_classifier.classify()`（`backend/app/services/extraction/document_classifier.py:69`）实现**本体驱动的可解释打分**：

1. 经 `engine.get_subclasses(RegulatoryDocument)` 枚举全部候选文档类型
2. 每个候选的信号词来自两处：rdfs:label 分词（通用）+ `_CURATED_SIGNALS`（强信号）
3. 对 haystack（标题 + 各级标题 + 前 12 段）子串计分，取最高分且过阈值者

CMCReport 已有 12 个人工强信号（CMC、原料药、备样生产、临床备样、合成路线、设备需求……），命中即加 2 分权重。

### 2.3 三阶段 NER 标注管线

```
Stage-1: GLiNER span 检测（CJK 字符级分词，009 根因修复）
  └─ _CJKAwareWordsSplitter：每个汉字单独成词，使 GLiNER 在字级粒度枚举 span
Stage-2: bge-base-zh-v1.5 嵌入 → ~201 本体类余弦归类（阈值 0.50）
Stage-3: 类数据属性标签 → 属性三元组
```

产出 `ExtractionCandidate`（入 DB 待审核），前端通过 tiptap JSON 高亮预览。

### 2.4 关系抽取

`extract_relationships()`（`backend/app/services/extraction/relation_extractor.py:635`）：

```
DocStructure → classify → 反查对象属性 → 按 range 调 endpoint finder → 连边
```

**已有的 10 个 endpoint finders** 覆盖了风险评估报告所需的全部信息类型：

| finder | range 类 IRI | 风险评估报告中的信息 |
|--------|-------------|-------------------|
| `find_drug_product` | `DrugProduct` | HRS-1234 基本信息（分子式、剂型、PDE） |
| `find_synthesis_route` | `SynthesisRoute` | 合成路线（递归 SynthesisStep） |
| `find_equipment` | `Equipment` | 使用设备清单（编号、名称、规格、材质） |
| `find_safety_risk` | `SafetyRisk` | 安全风险（人员/设备/物料/文件/三废） |
| `find_quality_risk` | `QualityRisk` | 质量风险 |
| `find_cleaning` | `CleaningProcess` | 设备清洗方法 |
| `find_residue` | `Residue` | 清洁残留物（溶解度数据） |
| `find_shared_line` | `SharedLineAssessmentData` | 共线评估（PDE、NOAEL、HED、安全系数） |
| `find_storage` | `StorageCondition` | 存放条件（温度、避光、有效期） |
| `find_degradation` | `DegradationPathway` | 降解途径（条件、降解率、杂质） |

每条 edge 携带 `source_ref`（如 `"表 设备清单"`、`"§ 合成路线"`），支持源文档回溯。

### 2.5 推理引擎

`evaluate()`（`backend/app/services/reasoning/interpreter.py:160`）接受 `Facts` 对象，支持的谓词操作：

| 操作 | 语义 | 风险评估场景映射 |
|------|------|----------------|
| `some_values_from` | ∃ property . filler_class | "使用的设备中是否存在搪玻璃反应釜" |
| `class_membership` | 对象属于某类 | "HRS-1234 是否属于高活性药物" |
| `datatype_facet` | 数值比较 (gt/lt/eq...) | "PDE < 10μg → 高活性" |
| `boolean_has_value` | 布尔判断 | "是否需要灭活" |
| `literal_eq` / `literal_cmp` | 字面值/数值比较 | "给药途径 == 口服" |
| `class_present` | 药品分类存在性 | "是否为头孢类/青霉素类" |
| `external_alignment` | 外部分类对齐 | ChEBI/ATC 分类匹配 |

推理结果为三值逻辑（TRUE / FALSE / UNKNOWN），支持开放世界假设（OWA）。

## 4. 需增量实现的三层

### 4.1 桥接层：edges → Facts（~200 行）

**位置**: `backend/app/services/reasoning/fact_bridge.py`（新建）

**职责**: 从 `extract_relationships()` 返回的 edges 列表构造临时 `Facts` 对象，**免入库即可跑推理规则**（"预览推理"路径）。

```python
def edges_to_facts(edges: list[dict], doc_class: dict | None) -> Facts:
    """从关系抽取边列表构造推理引擎所需的 Facts。

    不写入 DB，不创建个体——纯内存转换，使推理引擎可对尚未审核入库的
    抽取结果直接跑 DecisionRule 评估。
    """
    relations: dict[str, list[str]] = {}
    data_values: dict[str, Any] = {}
    scalars: dict[str, Any] = {}

    for edge in edges:
        pred_iri = edge["predicate_iri"]
        obj_class = edge["object_class_iri"]
        obj_text = edge["object_text"]

        # 对象属性 → relations（推理用 class_membership / some_values_from）
        relations.setdefault(pred_iri, []).append(obj_class)

        # 数据属性 → data_values / scalars（推理用 datatype_facet / literal_eq）
        for dp in edge.get("object_data_properties", []):
            dp_iri = dp.get("iri")
            dp_value = dp.get("value")
            if dp_iri and dp_value:
                data_values[dp_iri] = dp_value
            if dp.get("label"):
                scalars[dp["label"]] = dp_value

    # 从 DrugProduct 端点提取药品分类标签
    drug_classes = set()
    for edge in edges:
        if "DrugProduct" in edge.get("object_class_iri", ""):
            for dp in edge.get("object_data_properties", []):
                if "分类" in dp.get("label", "") or "类别" in dp.get("label", ""):
                    drug_classes.add(dp["value"])

    return Facts(
        relations=relations,
        data_values=data_values,
        scalars=scalars,
        drug_classes=drug_classes,
        alignments={},
    )
```

**设计决策**:
- 不创建 A-Box 个体：风险评估报告是"给定文档内容、即时推导"的场景，不应副作用入库
- 与 `ActionEngine` 的入库推理路径并行，复用同一 `evaluate()` 引擎
- `source_ref` 保留在 edges 中透传到报告层，不进 Facts（Facts 只管推理谓词）

### 4.2 报告模板层：推理结果 → 报告结构（~300 行）

**位置**: `backend/app/services/reporting/risk_report_generator.py`（新建）

**职责**: 将推理结果映射到 QS-A-020F05 报告结构。

```python
@dataclass
class RiskRow:
    """Assessment 评估过程表的一行。"""
    hazid: str                    # 风险类型（人员/生产设备/物料管理/文件/三废处理）
    contributing_factors: str     # 风险因素
    pre_control_level: str        # 控制前风险水平（高/中/低）
    post_control_level: str       # 控制后风险水平
    control_measures: str         # 风险控制措施
    traceability: str             # 控制措施可追溯性
    status: str                   # 风险状态（可以接受/不可接受）

@dataclass
class EquipmentEntry:
    """设备清单表的一行。"""
    seq: int
    equipment_id: str             # 设备编号（RE64202）
    name: str                     # 设备名称（搪玻璃反应釜）
    spec: str                     # 规格（500L）
    material: str                 # 材质（搪玻璃）

@dataclass
class RiskReport:
    """QS-A-020F05 风险评估表的完整结构。"""
    doc_no: str = "QS-A-020F05"
    revision: str = ""
    effective_date: str = ""

    # SECTION I
    subject_description: str = ""          # 风险评估对象基本描述
    equipment_tables: dict[str, list[EquipmentEntry]] = field(default_factory=dict)
    equipment_notes: list[str] = field(default_factory=list)
    team_members: list[dict] = field(default_factory=list)

    # Assessment
    assessment_rows: list[RiskRow] = field(default_factory=list)

    # 签批
    qa_comments: str = ""
    approvers: list[dict] = field(default_factory=list)

    # SECTION II
    risk_review: str = ""
    conclusion: str = ""
```

报告生成逻辑：

```python
class RiskReportGenerator:
    def generate(
        self,
        edges: list[dict],
        doc_class: dict | None,
        doc_structure: DocStructure,
        engine: OntologyEngine,
    ) -> RiskReport:
        report = RiskReport()

        # 1. SECTION I — 基本描述：从 DrugProduct 端点拼装
        report.subject_description = self._build_subject_description(
            edges, doc_structure
        )

        # 2. 设备清单：按车间分组
        report.equipment_tables = self._build_equipment_tables(edges)

        # 3. Assessment — 评估过程
        facts = edges_to_facts(edges, doc_class)
        rules = engine.list_decision_rules(rule_group="risk_assessment")

        for rule in self._sort_by_hazid(rules):
            pre_level = self._evaluate_pre_control(rule, facts)
            post_level = self._evaluate_post_control(rule, facts)
            report.assessment_rows.append(RiskRow(
                hazid=rule.category,
                contributing_factors=rule.description,
                pre_control_level=pre_level,
                post_control_level=post_level,
                control_measures=rule.control_measure,
                traceability=rule.traceability_docs,
                status="可以接受" if post_level in ("低",) else "不可接受",
            ))

        return report

    def _evaluate_pre_control(self, rule, facts: Facts) -> str:
        """控制前风险水平：直接对当前事实集跑 pattern。"""
        result = evaluate(rule.pattern, facts)
        return rule.conclusion if result == TRUE else "低"

    def _evaluate_post_control(self, rule, facts: Facts) -> str:
        """控制后风险水平：假设控制措施已实施，注入 postcondition 后重新推理。
        与 ActionEngine 的 action precondition/postcondition 模式对齐。"""
        augmented = self._apply_postconditions(facts, rule.postconditions)
        result = evaluate(rule.pattern, augmented)
        return "低" if result != TRUE else rule.conclusion
```

**设计决策**:
- 每个 HazID 维度对应一组 `DecisionRule`（通过 `rule_group="risk_assessment"` + `category` 字段区分人员/设备/物料/文件/三废）
- 控制前风险 = 对原始 Facts 的 `evaluate()` 结果映射到风险等级
- 控制后风险 = 假设 `postconditions` 已实施后的二次 `evaluate()`，与 `ActionEngine` 的 precondition/postcondition 机制天然对齐
- `source_ref` 从 edges 透传到报告中的可追溯性列

### 4.3 渲染层：报告结构 → .docx 文件（~200 行）

**位置**: `backend/app/services/reporting/docx_renderer.py`（新建）

**方案选择**:

| 方案 | 优劣 | 适用 |
|------|------|------|
| python-docx 生成 .docx | 已有依赖、与输入同格式、合规审计无转换损耗 | **推荐** |
| Jinja2 → HTML → PDF | 需新增 WeasyPrint 依赖、气隙部署多一步 | 备选 |
| JSON → 前端渲染 | 在线预览好、但不产生可存档文件 | 补充（预览用） |

推荐 **python-docx**——项目已依赖、与输入格式一致、合规审计认可。

```python
def render_risk_report(report: RiskReport, output_path: Path) -> Path:
    """将 RiskReport 结构渲染为 QS-A-020F05 格式的 .docx 文件。"""
    doc = Document()

    # 页眉：文件号 + 修订号 + 生效日期
    _add_header(doc, report)

    # SECTION I — 风险评估对象基本描述
    doc.add_heading("SECTION I 部分 1", level=1)
    doc.add_paragraph(report.subject_description)

    # 设备清单表（按车间分组）
    for workshop, entries in report.equipment_tables.items():
        doc.add_paragraph(f"HRS-1234 使用 {workshop} 车间设备见下表：")
        table = doc.add_table(rows=1, cols=5)
        table.style = "Table Grid"
        _set_headers(table, ["序号", "设备编号", "设备名称", "规格", "材质"])
        for entry in entries:
            row = table.add_row().cells
            row[0].text = str(entry.seq)
            row[1].text = entry.equipment_id
            row[2].text = entry.name
            row[3].text = entry.spec
            row[4].text = entry.material

    # 评估小组
    _add_team_table(doc, report.team_members)

    # Assessment — 评估过程
    doc.add_heading("Assessment 评估过程", level=2)
    assess_table = doc.add_table(rows=1, cols=7)
    assess_table.style = "Table Grid"
    _set_headers(assess_table, [
        "HazID\n风险类型", "Contributing Factors\n风险因素",
        "Risk level before\n控制前风险水平", "Risk level after\n控制后风险水平",
        "Risk Control Measure(s)\n风险控制措施",
        "Traceability\n控制措施可追溯性", "Status\n风险状态",
    ])
    for row_data in report.assessment_rows:
        row = assess_table.add_row().cells
        row[0].text = row_data.hazid
        row[1].text = row_data.contributing_factors
        row[2].text = row_data.pre_control_level
        row[3].text = row_data.post_control_level
        row[4].text = row_data.control_measures
        row[5].text = row_data.traceability
        row[6].text = row_data.status

    # QA 意见 + 签批
    _add_approval_section(doc, report)

    # SECTION II — 风险回顾
    doc.add_heading("SECTION II 部分 2", level=1)
    doc.add_paragraph(report.risk_review or "[待填]")
    doc.add_paragraph(f"Conclusion 结论：\n{report.conclusion or '[待填]'}")

    doc.save(str(output_path))
    return output_path
```

## 6. 推理逻辑详解

### 6.1 控制前风险水平推导

| 事实条件 | 操作 | 推导结果 |
|---------|------|---------|
| PDE < 10μg（高活性） | `datatype_facet(pde, lt, 0.01)` | 设备/物料 → 高风险 |
| 存在共线生产设备 | `some_values_from(usesEquipment, Equipment)` | 设备 → 高风险 |
| 细胞毒性药物 | `boolean_has_value(is_cytotoxic, true)` | 全维度 → 高风险 |
| 新工艺（无 SOP） | `some_values_from(hasSynthesisRoute, SynthesisRoute)` | 文件 → 高风险 |
| 非高活性 + 非高毒性 | 上述条件均不满足 | 各维度 → 低/中风险 |

以 HRS-1234 为例：PDE = 1.80mg（> 10μg，非高活性）、非细胞毒性 → 人员/物料/三废为中-低风险；但有共线生产设备且新工艺 → 设备/文件为高风险（控制前）。

### 6.2 控制后风险水平推导

每条规则的 `postconditions` 描述"控制措施已实施"后的事实变更：

```python
# 示例：设备风险的控制后推理
postconditions = {
    "equipment_qualified": True,        # 设备已确认
    "shared_line_assessed": True,       # 共线评估已完成
    "cleaning_validated": True,         # 清洁确认已完成
}

# 注入 postconditions 后，重新 evaluate
augmented_facts = apply_postconditions(original_facts, postconditions)
# pattern 中的 "共线未评估" 条件不再满足 → 风险降至低
```

### 6.3 风险等级映射

```python

    "HighRisk": "高",
    "MediumRisk": "中",
    "LowRisk": "低",
}
```
